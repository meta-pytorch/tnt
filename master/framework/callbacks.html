Callbacks¶

class torchtnt.framework.callback.Callback[source]¶

A Callback is an optional extension that can be used to supplement your loop with additional functionality. Good candidates for such logic are ones that can be re-used across units. Callbacks are generally not intended for modeling code; this should go in your Unit. To write your own callback, subclass the Callback class and add your own code into the hooks.

Below is an example of a basic callback which prints a message at various points during execution.

from torchtnt.framework.callback import Callback
from torchtnt.framework.state import State
from torchtnt.framework.unit import TEvalUnit, TPredictUnit, TTrainUnit

class PrintingCallback(Callback):
    def on_train_start(self, state: State, unit: TTrainUnit) -> None:
        print("Starting training")

    def on_train_end(self, state: State, unit: TTrainUnit) -> None:
        print("Ending training")

    def on_eval_start(self, state: State, unit: TEvalUnit) -> None:
        print("Starting evaluation")

    def on_eval_end(self, state: State, unit: TEvalUnit) -> None:
        print("Ending evaluation")

    def on_predict_start(self, state: State, unit: TPredictUnit) -> None:
        print("Starting prediction")

    def on_predict_end(self, state: State, unit: TPredictUnit) -> None:
        print("Ending prediction")

To use a callback, instantiate the class and pass it in the callbacks parameter to the train(), evaluate(), predict(), or fit() entry point.

printing_callback = PrintingCallback()
train(train_unit, train_dataloader, callbacks=[printing_callback])

Built-in callbacks¶

We offer several pre-written callbacks which are ready to be used out of the box:

BaseCSVWriter(header_row, dir_path[, ...]) A callback to write prediction outputs to a CSV file.
EarlyStopping(monitored_attr, early_stop_checker) This callback checks the value of a monitored attribute on a Unit and stops the training if the value does not improve.
GarbageCollector(step_interval) A callback that performs periodic synchronous garbage collection.
IterationTimeLogger(logger, *[, ...]) A callback which logs iteration times as scalars to TensorBoard.
Lambda(*[, on_exception, on_train_start, ...]) A callback that accepts functions run during the training, evaluation, and prediction loops.
LearningRateMonitor(loggers, *[, ...]) A callback which logs learning rate of tracked optimizers and learning rate schedulers.
MemorySnapshot(*, memory_snapshot_profiler) A callback for memory snapshot collection during training, saving pickle files to the user-specified directory.
ModuleSummary(max_depth, process_fn, ...) A callback which generates and logs a summary of the modules.
PeriodicDistributedSync([sync_every_n_steps]) A callback to sync all distributed workers at a given frequency.
ProgressReporter() A simple callback which logs the progress at each loop start/end, epoch start/end and step start/end.
PyTorchProfiler(profiler) A callback which profiles user code using PyTorch Profiler.
SlowRankDetector(*[, check_every_n_steps, ...]) A callback which detects slow ranks every N steps/epochs by comparing the time on each process.
SystemResourcesMonitor(loggers, *[, ...]) A callback which logs system stats, including: - CPU usage - resident set size - GPU usage - cuda memory stats
TensorBoardParameterMonitor(logger) A callback which logs module parameters as histograms to TensorBoard.
TimeLimitInterrupter([duration, timestamp, ...]) This callback tracks the time spent in training and stops the training loop when a time limit is reached.
TimeWaitForBatchLogger(logger, *[, ...]) A callback which logs time wait for batch as scalars to a MetricLogger.
ThroughputLogger(logger, throughput_per_batch, *) A callback which logs the train/eval/predict/fit throughput. For instance, it can be used to log QPS and number of batches processed per second. The callback logs the throughput on a step basis and on an epoch basis. We measure the throughput by dividing the number of batches processed (times the number of items in batch) by the time it took to process the batch: On a step granularity, we do this by leveraging the already collected timers for the iteration time and data wait time. On an epoch granularity, we measure the time between on_train_epoch_start and on_train_epoch_end on this callback to calculate the throughput.
TorchSnapshotSaver(dirpath, *[, ...]) A callback which periodically saves the application state during training using TorchSnapshot.
TQDMProgressBar([refresh_rate, file]) A callback for progress bar visualization in training, evaluation, and prediction.
TrainProgressMonitor(loggers) A callback which logs training progress in terms of steps vs epochs.
